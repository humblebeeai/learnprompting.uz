# Rekursiv injektsiya

Avval ko'rganimizdek, prompting hacking'ga qarshi juda yaxshi himoya usuli bu â€” bir LLM'ning chiqishini boshqa bir LLM yordamida baholash va unda dushmanona chiqish bor-yo'qligini tekshirishdir. Ushbu himoyani _rekursiv injektsiya hujumi_ yordamida chetlab o'tish mumkin. Bu hujum birinchi LLM'ga shunday prompt kiritishni o'z ichiga oladiki, u chiqishda ikkinchi LLM uchun injektsiya ko'rsatmasini o'z ichiga oladi.

Keling, biz buzmoqchi bo'lgan prompt quyidagicha deb faraz qilamiz:

<AIInput>
Quyidagilar va tabiat haqida ijobiy narsa ayting:
</AIInput>

Ushbu promptning chiqishi quyidagi prompt yordamida baholanadi:

<AIOutput>
Quyidagida so'kinish yoki haqoratli mazmun bormi (HA yoki YO'Q)?
</AIOutput>

Ikkala promptni ham quyida ko'rsatilganidek promptga qo'shimcha qilib buzishimiz mumkin. E'tibor bering, bu ham o'zgaruvchi qiymatini tayinlash hujumi misolidir (@kang2023exploiting).
